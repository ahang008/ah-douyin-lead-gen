# 获客型抖音口播稿生产系统

## 设计哲学

这个 skill 的本质是一个**产品系统**，不是单一 prompt。它覆盖定位、建库、选题、成稿、审核、知识库回流、数据复盘。

系统边界大不是问题。真正的问题是：**大系统必须有轻入口、强恢复、模块化路由、渐进式资产积累。**

每次对话都要尽量做到三件事：
1. 当前建议你做什么
2. 基于现有信息产出了什么
3. 本轮新增了什么资产，下次可以少问什么

---

## 何时触发

**精准触发**（这些触发词指向获客系统）：
- `/douyin-lead-gen`、`/获客口播`
- 「帮我写获客稿」「获客型内容」
- 「让人主动私信的稿子」
- 「为什么这篇没人咨询」
- 「把这个选题改成能获客的口播」
- 「生成获客选题」「用行业选题连连看生成选题」

**弱化触发**（这些泛需求优先交给泛流量口播系统）：
- 「写口播」「写抖音口播稿」「帮我写口播稿」→ 默认走 `douyin-script-maker-v2`，除非用户明确表达"想让人私信/咨询/获客"

**不触发**：
- 纯娱乐/搞笑/蹭热点内容
- X 平台推文、小红书图文、公众号长文
- 明确不需要获客的泛流量口播内容

## 路由逻辑

```
用户触发 skill
  ↓
Context Boot：读取 .session-state.md
  ├─ 本窗口首次触发 → 展示完整"获客内容作战台"
  └─ 本窗口已展示过 → 只展示一句压缩状态
  ↓
自动判断用户请求 → 进入对应模块：
  ├─ 用户没有选题 → 模块 B：选题生成（行业选题连连看）
  ├─ 用户有普通选题 → 模块 C：选题转化
  ├─ 用户有选题 + 要成稿 → 模块 D：快速成稿
  ├─ 用户给旧稿要审核 → 模块 E：审核修改
  ├─ 用户要系统建库 → 模块 F：深度建库
  ├─ 用户给数据 → 模块 G：数据复盘
  ├─ 用户说"继续上次" → 跳到上次卡住的模块
  └─ 任何模块结束后 → 模块 H：知识库回流（静默执行）
```

## 模块 A：Context Boot（强制上下文启动）

**触发**：每次新上下文窗口第一次触发 skill 时强制执行。用户说"继续上次""启动获客系统""我现在做到哪了"时也强制执行。

**执行逻辑**：
1. 读取 `.session-state.md`
2. 判断当前对话窗口是否已经展示过 Context Boot（输出中标记 `[Context Boot 已展示：本窗口]`）
3. 如果本窗口第一次触发 → 展示完整"获客内容作战台"
4. 如果本窗口已展示过 → 只展示一句压缩状态，不重复刷屏
5. 展示后自动判断用户当前请求应该进入哪个模块

**关于"新上下文窗口"的识别**：
- 当前对话上下文中没有出现过 `[Context Boot 已展示：本窗口]` → 视为新窗口
- 用户说"继续上次""打开系统""启动获客系统""我现在做到哪了" → 强制展示
- 用户直接输入选题/旧稿/数据 → 先展示压缩恢复，再进入对应模块
- 同一窗口内已展示过则不重复完整展示，除非用户要求

**输出格式**：

```
[Context Boot 已展示：本窗口]

## 获客内容作战台

### 1. 当前定位
- 你卖什么：[从知识库提取]
- 目标客户：[从知识库提取]
- 核心购买理由：[从知识库提取]
- 行业公敌：[从知识库提取]
- 当前缺口：[本轮最缺什么——选题/稿子/数据/素材]

### 2. 上次进度
- 上次做到了：[模块 + 具体产出]
- 最近生成的选题：[列出]
- 最近生成的稿子：[列出 + 状态]
- 待处理事项：[列出]

### 3. 建议
我建议你现在进入：[模块名]

可选：
A. 生成选题（行业选题连连看）
B. 写一条获客稿
C. 审核旧稿
D. 补充知识库
E. 复盘数据
F. 继续上次
```

**注意**：不要问一堆开放问题。直接给建议。如果用户已经表达了明确任务，展示完作战台后直接进入任务，不要让用户再选择。

## 模块 B：选题生成（行业选题连连看）

**触发**：用户没有明确选题 / 用户说"今天拍什么" / "帮我生成获客选题" / 用户只有行业/产品/方向

**方法**：行业选题连连看 —— 一个 25 格选题矩阵

### 方法定义

**第一层：行业核心**
中间写用户的行业或产品（1个词）。例如：二手车、AI 自媒体、付费咨询、知识付费、本地生活。

**第二层：行业相关**
围绕行业写 8-12 个相关关键词。例如二手车：合同、公里数、电车、油车、年限、保险、事故、水泡、价格、车况、过户、贷款。

**第三层：人群/场景**
写 10-15 个会接触这个行业的人群或场景。人群例子：大学生、新手、情侣、宝妈、上班族、给父母买、结婚、小老板、刚毕业的人、预算有限的人。场景例子：抖音、小红书、朋友圈、4S店、瓜子、懂车帝、线下门店、社群。

### 组合方式

用"三点一线"连连看：**行业核心 + 行业相关 + 人群/场景**

例如：二手车 + 水泡 + 瓜子、二手车 + 保险 + 新手、二手车 + 公里数 + 女生

每个组合都可以变成一个选题方向。

### 生成流程

1. 识别用户行业/产品。如果行业不清楚，只问一句："你这条内容最终想为哪个产品或业务获客？"
2. 生成 1 个行业核心词
3. 生成 8-12 个行业相关词
4. 生成 10-15 个人群/场景词
5. 用三点一线组合出 20-30 个候选
6. 用获客筛选标准筛掉泛流量题
7. 最终输出 5 个最值得写的获客型选题

### 获客筛选标准

每个选题过以下筛子：
- 这个选题讲完后，观众最可能产生的咨询需求是什么？
- 这个需求是否直接指向你的产品/服务？
- 换个博主讲同样的选题，能不能讲出跟你一样的东西？

### 输出格式

```
## 行业选题连连看

### 行业核心
[行业/产品]

### 行业相关词（8-12个）
1. 2. 3. ...

### 人群/场景词（10-15个）
1. 2. 3. ...

### 组合出的选题池（部分示例）
1. [行业 + 相关词 + 人群/场景] → [选题方向]

### 最推荐的 5 个获客选题
1. 选题：[标题]
   目标客户：[谁会产生咨询需求]
   购买理由：[为什么看完想找你]
   为什么适合获客：[转化路径]
   可写角度：[2-3个]

2. ...
```

用户选定选题后 → 进入模块 D（快速成稿）。

### 备选：倒推选题法

如果用户已有较完整的知识库，也可使用倒推公式：
```
产品优势 → 变成行业标准 → 倒推客户焦虑 → 形成选题
```
详见 `data/02-选题公式与筛选标准.md`。

## 模块 C：选题转化

**触发**：用户已有普通选题 / 用户说"这个选题怎么改成获客型" / 用户给了一个想法但不够锋利

**做什么**：把普通选题改成获客型选题，输出不同角度版本。

**流程**：
1. 读取用户给的选题，理解核心观点
2. 追问一句："这个选题最终想为哪个产品/服务获客？"（如果用户说不清，直接帮他从选题中推断）
3. 输出 2-3 个获客型改写版本，每个版本标注：
   - 改写后的标题方向
   - 目标客户
   - 转化路径（观众看完 → 产生什么想法 → 想咨询什么 → 指向什么产品）
   - 跟原版的核心差异

**输出格式**：

```
## 选题转化

原选题：[用户原话]

### 版本1：[获客型标题]
- 目标客户：
- 转化路径：
- 跟原版的差异：

### 版本2：...
### 版本3：...
```

用户选定版本后 → 进入模块 D（快速成稿）。

## 模块 D：快速成稿

**触发**：用户说"写一条关于 X 的获客稿" / 用户给了选题 + 经历/旧稿 / 用户明确要成稿

**核心原则**：
- 不要求完整知识库。有就用，没有就问。
- 缺关键信息只问 1-3 个问题，不问开放式的"你有什么经历"。
- 分层读取：只读必要的文件，不默认加载全部 data。

**分层读取策略**：

| 层级 | 文件 | 何时读 |
|------|------|--------|
| 必读 | `Step3-口播稿生成.md` | 每次 |
| 必读 | `data/00-获客型文案核心框架.md` | 每次 |
| 选读 | `data/01-四种文案模板库.md` | 需要选模板结构时 |
| 选读 | `data/06-反常识与认知颠覆公式.md` | 需要反常识开头时 |
| 选读 | `data/10-数据驱动的口播设计.md` | 需要控制完播节奏时 |
| 选读 | `data/05-无用之用与购买理由.md` | 需要确定无用之用配比时 |
| 选读 | 知识库相关文件 | 按选题方向只读 1-2 个最相关的 |

**禁止默认读取全部 data 文件。禁止默认读取全部知识库。**

### 成稿流程

1. **确认关键信息**（如果知识库没有，只问 1-3 个问题）：
   - 这条内容为哪个产品/服务获客？
   - 有没有跟这个选题相关的真实经历？
   - 目标客户最怕什么/最想要什么？

2. **走获客型四步结构**：
   - 第一步：抛出反常识断言或行业问题（抢占定义权）
   - 第二步：否定旧模式（打碎现有选项，制造认知真空）
   - 第三步：建立新标准（展示可能性，保留完整交付）
   - 第四步：升华收尾（金句，不留CTA）

3. **应用硬约束**（见下方"硬约束"章节）

4. **输出口播稿**（1000-1400字） + 获客逻辑自检

**核心铁律**：展示价值判断，保留完整交付。
- 可以讲为什么、讲判断标准、讲底层逻辑、讲错误路径、讲案例
- 但不免费交付完整 SOP、工具配置、逐步执行清单
- 目标不是操控用户，而是区分免费内容和付费交付的边界

## 模块 E：审核修改

**触发**：用户给出旧稿 / 用户说"这篇能不能获客" / "帮我改到能拍"

**做什么**：审核获客链路，输出扣分项、修改建议、替换稿。

**分层读取**：

| 层级 | 文件 | 何时读 |
|------|------|--------|
| 必读 | `Step4-内容审核.md` | 每次 |
| 选读 | `data/03-论述逻辑自查框架.md` | 逻辑有问题时 |
| 选读 | `data/04-情绪注入与审核标准.md` | 情绪节奏有问题时 |
| 选读 | `data/10-数据驱动的口播设计.md` | 信息点密度有问题时 |

### 评分系统（修订版）

评分是诊断工具，不是预测结果。30 分以上不代表一定能获客，只代表结构具备发布条件。

**评分输出必须包含**：
- 结构分（5 维度 35 分制）
- 获客链路风险（指出最可能断掉的环节）
- 最大不确定性（评分无法覆盖的变量）
- 下一条内容要验证的假设

**样本不足声明**：不足 10 条历史样本前，不输出过度确定的预测。评分仅供参考，真正校准来自发布数据、评论质量、私信数量和私信内容。

### 评分维度

| 维度 | 分值 | 核心问题 |
|------|------|---------|
| 获客逻辑 | 10 | 看完会不会想找博主？展示价值判断而非教操作？不留CTA？ |
| 认知手术 | 8 | 旧认知是否被打破？新标准是否指向博主产品？ |
| 推力结构 | 7 | 每段推下一段？信息点密度够不够？ |
| 情绪曲线 | 5 | 高低交替？有无呼吸段？结尾赋能而非焦虑？ |
| 身份框架 | 5 | 语气是规则制定者还是推销员？ |

- 30-35：结构具备发布条件
- 25-29：修改可用
- 20-24：需重写
- <20：定位或素材有根本问题

用户可以选择让 AI 直接输出修改后的替换稿。

## 模块 F：深度建库

**触发**：用户说"帮我系统建库" / "我要长期做这个账号" / 用户明确愿意做完整定位梳理

**重要**：深度建库不是默认入口。第一次使用不需要完整知识库。系统应该在每次对话中顺手收集信息（见模块 H）。

只有当用户明确要做完整定位梳理时，才进入深度建库。

**流程**：逐个模块推进（定位声明 → 产品优势 → 客户画像 → 行业公敌 → 个人经历 → 购买理由 → 选题词根），每个模块展示给用户确认。

**产出 7 个文件**到 `✍️ 写作系统/抖音/03-获客知识库/`：
- `00-定位声明.md`
- `01-产品优势清单.md`
- `02-客户画像.md`
- `03-行业公敌与排他标准.md`
- `04-个人经历库.md`
- `05-购买理由.md`
- `06-选题词根库.md`

**关键约束**：AI 不编造任何内容，所有素材来自用户口述。

详细流程见 `Step1-定位与知识库初始化.md`、`data/09-定位与人设公式.md`。

## 模块 G：数据复盘

**触发**：用户说"数据回来了" / 用户给播放量、点赞、收藏、评论、私信 / 用户问"为什么有人看没人私信"

**做什么**：把平台数据翻译成获客逻辑诊断，更新下一条选题/稿子的策略。

**流程**：
1. 读取 `Step6-数据回收与校准.md` + `data/11-数据回收信号映射表.md`
2. 收集六个平台指标 + 私信内容
3. 逐指标对照映射表诊断获客逻辑
4. 输出校准报告 + 知识库更新建议

**关键视角**：数据不用于评判稿子好坏，用于诊断获客逻辑哪里断了。每个数字背后是一个获客假设的验证或证伪。

## 模块 H：知识库回流

**触发**：每次选题、写稿、审核、复盘结束后静默执行 / 用户说"记录一下"

**做什么**：从本轮对话中提取新增信息，回写知识库。不打断主任务。输出简短"本轮新增资产"。

**提取规则**：

| 本轮出现 | 回流到 | 判断标准 |
|---------|--------|---------|
| 新的个人经历/案例 | `04-个人经历库.md` | 之前知识库没有的经历 |
| 新的客户痛点 | `02-客户画像.md` | 之前没记录过的客户焦虑 |
| 新的产品优势 | `01-产品优势清单.md` | 之前没记录的优势 |
| 新的选题词根 | `06-选题词根库.md` | 使用了新的词根 |
| 新的购买理由/金句 | `05-购买理由.md` | 用户表达了新的价值主张 |
| 新的反常识观点 | `03-行业公敌与排他标准.md` | 用户表达了新的行业判断 |

**输出格式**：

```
## 本轮新增资产
- [文件]：新增了 [内容摘要]
- （如无新增）本轮无新增资产
```

详细流程见 `Step5-知识库维护与回流.md`。

## 知识库策略：边用边长

保留 7 个知识库文件，但不作为首次使用门槛。

第一次使用可以没有知识库。系统在每次对话中顺手收集：
- 用户卖什么
- 目标客户是谁
- 用户真实经历
- 客户痛点
- 购买理由
- 行业公敌
- 反常识观点
- 口播表达偏好
- 发布后的数据反馈
- 私信内容和客户问题

每次任务结束后，通过模块 H 把新增信息归档到对应文件。

完整知识库路径：
```
✍️ 写作系统/抖音/03-获客知识库/
  00-定位声明.md
  01-产品优势清单.md
  02-客户画像.md
  03-行业公敌与排他标准.md
  04-个人经历库.md
  05-购买理由.md
  06-选题词根库.md
  .session-state.md          ← 跨会话持久状态（自动维护）
```

## 硬约束

### 获客边界

1. **展示价值判断，保留完整交付**：可以讲为什么、判断标准、底层逻辑、错误路径、案例。但不免费交付完整 SOP、工具配置、逐步执行清单。目标是区分免费内容和付费交付的边界，不是操控用户。
2. **禁止在结尾放 CTA**：不写"私信我""评论区扣1""加我微信"
3. **禁止推销员语气**：不说"我们产品很好""欢迎咨询"
4. **禁止制造痛苦不收束**：结尾必须让用户感到被赋能，不是焦虑
5. **禁止天上掉馅饼**：暗示轻松必须配上杠杆解释

### 口播稿标准

6. **高频换信息点**：不在一个点上讲深，每 4-5 秒一个新信息点（800 字稿 >= 10 个信息点）
7. **推力结构**：每段推着观众往下一段走，不是并列罗列
8. **七种切换技法至少用四种**：认知层级/视角/时间轴/情绪温度/颗粒度/论证模式/对话感。独白超过 15 秒必须插入对话互动。
9. **排比最多一次**：全片排比结构最多用一次
10. **高密度后有呼吸段**：连续信息轰炸后要有缓和段

### 素材使用

11. **不编造经历**：知识库里没有的素材，停下来问用户，不自己编
12. **不编造数据**：不造"我有一个学员叫小王"

### 跨会话持久化

13. **自动收集状态**：每次对话结束前，或用户说"回头/下次/先这样/我想想"时，AI 自动将本轮关键信息写入 `.session-state.md`
14. **Context Boot 优先**：每次 skill 触发时先展示作战台，再分流到具体模块
