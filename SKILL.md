# 获客型抖音口播稿生产系统

## When to use

**手动触发**：`/douyin-lead-gen` 或用户说"帮我写获客稿""获客型内容""生成获客选题""建获客知识库"

**适用场景**：
- 想做能带来客户咨询的抖音口播内容
- 不想做泛流量，想做精准获客
- 已有的口播稿"讲得挺好但没人来找我"
- 想系统性地建立自己的获客内容生产闭环

**不要触发**：
- 纯娱乐/搞笑/蹭热点内容
- X平台推文、小红书图文、公众号长文
- 不需要获客的泛流量口播内容（用 `/douyin-script-maker-v2`）

## 五阶段工作流

### Phase 1：定位与知识库初始化

**触发**：用户首次使用，知识库 `✍️ 写作系统/抖音/03-获客知识库/` 不存在或为空

**做什么**：带用户完成"自我挖掘"对话，产出 7 个知识库文件

**流程**：
1. 读取 `Step1-定位与知识库初始化.md` + `data/09-定位与人设公式.md`
2. 逐个模块推进：定位声明 → 产品优势 → 客户画像 → 行业公敌 → 个人经历 → 购买理由 → 选题词根
3. 每个模块展示给用户确认
4. 全部完成后输出"获客定位摘要"

**产出文件**：
- `00-定位声明.md`
- `01-产品优势清单.md`
- `02-客户画像.md`
- `03-行业公敌与排他标准.md`
- `04-个人经历库.md`
- `05-购买理由.md`
- `06-选题词根库.md`

**关键约束**：AI 不编造任何内容，所有素材来自用户口述。

---

### Phase 2：选题生成与筛选

**触发**：用户说"生成选题"/"找选题"/"今天拍什么"

**做什么**：从知识库倒推客户焦虑，生成 5-8 个选题，过双重筛选

**流程**：
1. 读取 `Step2-选题生成与筛选.md` + `data/02-选题公式与筛选标准.md` + `data/06-反常识与认知颠覆公式.md`
2. 读取全部知识库文件
3. 从产品优势倒推 → 客户焦虑 → 生成选题
4. 每个选题标注：公式来源/矩阵角色/常青热点/目标情绪
5. 过双重筛选（四问 + DELIVERR）
6. 输出 3-5 条通过筛选的选题
7. 用户选定 → Phase 3

**可选**：用户自带选题想法，AI 帮转化成获客型选题。

---

### Phase 3：口播稿生成

**触发**：Phase 2 选定选题后自动进入，或用户说"写稿"/"写这条"

**做什么**：基于选题+知识库素材，走四步结构生成 1000-1400 字获客型口播稿

**流程**：
1. 读取 `Step3-口播稿生成.md` + data/00 + data/01 + data/05 + data/06
2. 从知识库调取相关个人经历和案例
3. 确定矩阵角色 → 选择对应模板（A/B/C/D）
4. 走获客型四步结构：破题 → 否定旧模式 → 建立新标准 → 升华收尾
5. 应用硬约束（推力结构/无用之用配比/展示可能性隐藏细节）
6. 输出完整口播稿 + 获客逻辑自检

**核心铁律**：展示可能性，隐藏操作细节。

---

### Phase 4：内容审核

**触发**：Phase 3 成稿后自动进入，或用户说"审核"/"检查这篇"

**做什么**：逐段情绪标注 + 7项获客逻辑自查 + 量化评分

**流程**：
1. 读取 `Step4-内容审核.md` + data/03 + data/04
2. 逐段标注情绪等级（L1/L2/L3）
3. 过 7 项获客逻辑自查
4. 5 维度 35 分制量化评分
5. 输出审核报告

**评分标准**：

| 维度 | 分值 |
|------|------|
| 获客逻辑 | 10分 |
| 认知手术 | 8分 |
| 推力结构 | 7分 |
| 情绪曲线 | 5分 |
| 身份框架 | 5分 |
| **总分** | **35分** |

- 30-35：拍摄级 → Phase 5
- 25-29：修改可用 → 针对性修改
- 20-24：重写级 → 回 Phase 3
- <20：回炉级 → 回 Phase 1

---

### Phase 5：知识库维护与回流

**触发**：Phase 4 审核通过后自动执行，或用户说"记录一下"

**做什么**：从稿子中提取新信息，回写知识库

**流程**：
1. 读取 `Step5-知识库维护与回流.md`
2. 自动提取新经历/新痛点/新优势/新词根
3. 输出变更摘要
4. 询问用户是否有补充
5. 写入对应知识库文件

**补充功能**：发布后数据复盘（播放量/点赞/评论/私信分析）。

## 四种工作模式

本 Skill 提供 4 种显式模式，用户触发时选择：

| 模式 | 触发方式 | 走哪些 Phase | 适用场景 |
|------|---------|-------------|---------|
| **模式A：全流程** | "生成获客选题" / "帮我写一条获客稿" | Phase 1→2→3→4→5 或 2→3→4→5 | 不知道今天拍什么，从选题开始 |
| **模式B：快速成稿** | "写一条关于[选题]的获客稿" | Phase 3→4→5 | 已有明确选题，直接写稿 |
| **模式C：纯审核** | "审核这篇口播稿" / "帮我看看能不能获客" | 仅 Phase 4 | 已有成稿，只需审核和修改建议 |
| **模式D：维护知识库** | "更新我的知识库" / "记录一下[新经历]" | 仅 Phase 5 | 不写新稿，只补充/更新知识库 |
| **模式E：数据回收** | "数据回来了" / "复盘这条" / "这条稿子的数据" | Phase 6 | 稿子发布后回收数据并校准知识库 |

---

### Phase 6：数据回收与校准

**触发**：用户说"数据回来了"/"复盘这条"/"这条稿子的数据"

**做什么**：回收发布后数据，翻译成获客诊断，更新知识库

**流程**：
1. 读取 `Step6-数据回收与校准.md` + `data/11-数据回收信号映射表.md`
2. 从稿子追踪表定位目标稿子
3. 收集六个平台指标 + 私信内容
4. 逐指标对照映射表诊断获客逻辑
5. 输出校准报告 + 知识库更新建议
6. 更新 `.session-state.md`

### Phase 0：会话恢复

**触发**：每次 skill 被调用时，在任何其他 Phase 之前自动执行。

**做什么**：读取 `.session-state.md`，展示恢复摘要，让用户无缝衔接上一次的工作。

**流程**：
1. 读取 `Step0-会话恢复.md`
2. 检查状态文件 → 展示摘要 + 待处理 + 稿子追踪 + 逾期提醒
3. 询问用户：继续上次 / 处理待办 / 回收数据 / 新任务
4. 路由到对应 Phase

### Phase 0.5：自动状态收集

**触发**：每次对话结束前，或用户表达"回头再.../先这样/下次再..."时，自动执行。

**做什么**：无需用户操作，AI 自动将本轮对话的关键信息写入 `.session-state.md`：
- 本轮做了什么 Phase
- 产生了什么文件/决策
- 用户说了"回头给我/我想想/下次再补充"的内容 → 自动加入待处理队列
- 更新最后活跃时间戳

## 硬约束

### 来自获客型白皮书的核心禁令

1. **禁止在结尾放CTA**：不写"私信我""评论区扣1""加我微信"
2. **禁止教操作步骤**：不给SOP、不教工具配置、不让用户能自己复制
3. **禁止推销员语气**：不说"我们产品很好""欢迎咨询"
4. **禁止制造痛苦不收束**：结尾必须让用户感到被赋能，不是焦虑
5. **禁止天上掉馅饼**：暗示轻松必须配上杠杆解释

### 来自 CLAUDE.md 的口播稿标准

6. **高频换信息点**：不在一个点上讲深，每4-5秒一个新信息点（800字稿≥10个信息点）
7. **推力结构**：每段推着观众往下一段走，不是并列罗列
8. **七种切换技法至少用四种**：认知层级/视角/时间轴/情绪温度/颗粒度/论证模式/对话感，不给用户连续同一模式。独白超过15秒必须插入对话互动。
9. **排比最多一次**：全片排比结构最多用一次
10. **高密度后有呼吸段**：连续信息轰炸后要有缓和段

### 素材使用

11. **不编造经历**：知识库里没有的素材，停下来问用户，不自己编
12. **不编造数据**：不造"我有一个学员叫小王"

### 跨会话持久化

13. **自动收集状态**：每次对话结束前，或用户说"回头/下次/先这样/我想想"时，AI 自动将本轮关键信息写入 `.session-state.md`——做了什么Phase、产生了什么决策、有什么待处理。不需要用户提醒。
14. **会话恢复优先**：每次 skill 触发时先走 Phase 0 读取状态文件，展示待处理和逾期提醒。
