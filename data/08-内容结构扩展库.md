## 一、9种通用内容结构

当获客型四个核心模板不够用时，从以下扩展库中选择：

| 结构类型 | 公式 | 适用场景 |
|----------|------|----------|
| 观点+事例型 | 核心观点 → 多个故事印证 → 升华 | 生活感悟、社会观察 |
| 大观点+小观点型 | 打破常识的大观点 → 多个小观点论述 | 认知颠覆类、热点评论 |
| 干货教学型 | 问题场景 → 步骤拆解 → 成果展示 | 知识付费、技能培训 |
| 痛点共鸣型 | 痛点描述 → 情绪共鸣 → 解决方案 | 消费决策、情感内容 |
| 悬念反转型 | 反常事件 → 悬念留白 → 反转结局 | 搞笑、情感、剧情 |
| 对比反差型 | 通过对比展现差异 → 引发兴趣 | 测评、种草、前后对比 |
| 挑战互动型 | 规则说明 → 示范动作 → 点名互动 | 美妆、健身、美食 |

**注意**：获客型内容优先使用 data/01 的四个模板。扩展库里的结构只在默认模板不适用时作为替代。

## 二、推力结构是底层技法，不是独立结构类型

推力结构（悬念递进、信息密度控制、情绪波动）可以叠加到任何结构上：

- 观点+事例型 → 在每个事例之间用悬念递进衔接
- 干货教学型 → 用信息密度控制保证每15秒有新信息点
- 痛点共鸣型 → 用情绪波动法保持节奏

## 三、5步逆向拆解法（分析对标用）

当用户提供对标内容时，用它拆解：

1. **扫描整体框架**：诉求目标/目标受众/核心逻辑
2. **提取关键模块**：开头（钩子）→ 中间（证据/优势）→ 结尾（CTA/升华）
3. **模式匹配**：把拆出的模块跟已知结构对比
4. **处理混合结构**：逐段标注功能标签，找出主导模式
5. **验证**：读出声检查流畅度，问这个结构是否解决了用户的问题

## 四、开头钩子四维拆解法

拿到对标开头，拆四样东西：
1. 她说了什么？（字面内容）
2. 她用的是哪种钩子类型？（提问/反常识/痛点/数字冲击/悬念）
3. 她针对的是哪种情绪？（焦虑/好奇/共鸣/渴望）
4. 她用的句式是什么？（能不能抽象成一个公式？）

**示例**：
- 原文："我工作五年，攒下的钱还不如我同学炒一次基金"
- 钩子类型 = 反差对比
- 情绪 = 共鸣 + 轻微焦虑
- 句式公式 = `「我付出XX，却比不上别人XX」`

同一个博主通常只反复用2-3种开头公式，拆出来就是可复用的模板。

## 五、口播稿7步创作流程（总览）

```
对标导出 → Step1选题确认 → Step2需求验证 → Step3角度切入 
→ Step4标题拟定 → Step5框架搭建 → Step6内容填充 → Step7口播稿打磨
```

本 skill 的 Phase 2 对应 Step1-3，Phase 3 对应 Step4-7。
