# 获客型抖音口播稿生产系统

一个 Claude Code Skill，用 AI 帮你系统性地生产能带来客户咨询的抖音口播内容。

## 这是什么

不是泛流量口播稿生成器。这个 Skill 专做一件事：**让观众看完你的视频后主动来找你开口说话**。

核心逻辑来自 13 章获客型文案白皮书（6万字方法论），所有内容基于**你的真实经历和产品**，不编造、不套模板。

## 安装

### 方式一：Claude Code 插件市场（推荐）

```bash
claude plugins install @zephyr/douyin-lead-gen
```

### 方式二：手动安装

```bash
git clone https://github.com/ahang008/ah-douyin-lead-gen.git
cp -r ah-douyin-lead-gen ~/.claude/skills/
```

### 方式三：项目级安装

```bash
git clone https://github.com/ahang008/ah-douyin-lead-gen.git
cp -r ah-douyin-lead-gen /your-project/.claude/skills/
```

## 触发方式

在 Claude Code 中输入以下任意一种：

- `/douyin-lead-gen`
- `/获客口播`
- 「帮我写获客稿」
- 「获客型内容」
- 「生成获客选题」
- 「建获客知识库」
- 「审核这篇口播」

## 五种工作模式

| 模式 | 触发词 | 适用场景 |
|------|--------|---------|
| 全流程 | "生成获客选题" | 不知道拍什么，从选题到成稿全自动 |
| 快速成稿 | "写一条关于[选题]的获客稿" | 已有选题，直接写稿 |
| 纯审核 | "审核这篇口播稿" | 已有成稿，只需诊断 |
| 维护知识库 | "更新我的知识库" | 补充个人经历和产品素材 |
| 数据回收 | "数据回来了" | 发布后回收数据进行校准 |

## 工作流程

```
Phase 0: 会话恢复 → 展示上次进度，无缝衔接
Phase 1: 定位与知识库初始化 → 7个文件建立你的获客定位
Phase 2: 选题生成与筛选 → 从知识库倒推客户焦虑，过双重筛选
Phase 3: 口播稿生成 → 四步结构写获客型口播稿
Phase 4: 内容审核 → 5维度35分制量化评分
Phase 5: 知识库回流 → 自动提取新素材回写知识库
Phase 6: 数据回收与校准 → 发布后回收数据诊断获客逻辑
```

## 核心铁律

- **展示可能性，隐藏操作细节** — 告诉观众能做成什么样，不教怎么做
- **禁止在结尾放 CTA** — 不写"私信我""加我微信"
- **不编造经历** — 知识库里没有的素材，停下来问用户
- **禁止推销员语气** — 你是行业规则的制定者，不是卖货的
